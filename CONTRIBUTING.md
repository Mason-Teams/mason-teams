# Contributing to MASON

Thanks for your interest in making MASON better. Here's how you can help.

## Reporting Bugs

Found something broken? [Open a bug report](https://github.com/Mason-Teams/mason-teams/issues/new?template=bug_report.md). The more detail you include, the faster we can fix it:

- **Steps to reproduce** — Walk us through exactly what you did
- **Screenshots** — Especially helpful for UI issues or unexpected behavior
- **Logs** — Run `./scripts/masonctl logs --tail=100` and include the output
- **Environment** — OS, Docker version, architecture, browser (see the template)

Good bug reports save everyone time. "It doesn't work" is hard to act on. "When I run masonctl start on Ubuntu 24.04 with Docker 27.1, the dashboard shows a blank page — here's the console error and logs" is something we can fix.

## Suggesting Features

Have an idea? [Open a feature request](https://github.com/Mason-Teams/mason-teams/issues/new?template=feature_request.md). Tell us what problem it solves and how you'd like it to work.

## Pull Requests

We welcome PRs for bug fixes and documentation improvements. For anything larger, please open an issue first so we can discuss the approach before you invest time in it.

### Guidelines

- Keep PRs focused — one fix or improvement per PR
- Write clear commit messages that explain *why*, not just *what*
- Test your changes locally before submitting
- Update documentation if your change affects user-facing behavior

## Code of Conduct

Be kind, be constructive, be patient. This is a small project and we're doing our best. Disrespectful or unconstructive behavior won't be tolerated.

## Questions?

If you're not sure whether something is a bug or you need help getting set up, [open an issue](https://github.com/Mason-Teams/mason-teams/issues) anyway — we'd rather hear from you than have you stuck.

You can also reach us at [info@masonteams.com](mailto:info@masonteams.com).
